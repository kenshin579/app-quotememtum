# 명언 줄바꿈 시 단어 분리 방지 PRD

## 1. 현재 상태 분석

### 문제 현상

새 탭에 표시되는 명언이 길어 여러 줄로 줄바꿈될 때, 한국어 단어(어절)가 음절 단위로 잘려서 다음 줄로 넘어가는 문제 발생.

**예시 (스크린샷 기준)**:

```
"만약 내 작품에 메시지가 들어 있다면 그것은 다
름을 인정해도 좋다는 것이다. 그건 다른 말로 남
과 달라도 괜찮다는 것, 외모가 다르고 행동이 다
르고 말하는 방식도 다른 다른 색을 가진 누군가
를 판단하기 앞서 스스로에게 물어야 한다는 것이
다."
```

- `다름을` → `다` / `름을`
- `남과` → `남` / `과`
- `다르고` → `다` / `르고`
- `누군가를` → `누군가` / `를`
- `것이다` → `것이` / `다`

### 원인 분석

- `src/components/Quote.tsx`에서 명언을 렌더링하는 `<p>` 요소에 `word-break` 관련 CSS 속성이 지정되어 있지 않음.
- 브라우저 기본값(`word-break: normal`)은 CJK(한·중·일) 문자를 단어 경계로 보지 않고, 줄 끝에서 임의의 글자 사이를 끊을 수 있음.
- 결과적으로 어절 중간이 끊어져 가독성과 시각적 완성도가 떨어짐.

### 영향 범위

| 컴포넌트 | 위치 | 영향 |
|---------|------|------|
| `Quote.tsx` (본문) | `src/components/Quote.tsx:37` — `<p className="font-light drop-shadow-lg max-w-3xl">` | 명언 본문이 길어질 때 단어 분리 발생 |
| `Quote.tsx` (저자) | `src/components/Quote.tsx:52` — `<p className="mt-4 text-xl ...">` | 저자명이 길어질 가능성은 낮으나 동일 이슈 잠재 |

## 2. 채택 방안: `word-break: keep-all` 적용

명언 본문 `<p>` 요소에 CSS `word-break: keep-all` 속성을 적용하여, 한국어 어절(공백 단위)이 깨지지 않도록 한다.

### 동작 원리

| 속성 값 | 동작 |
|---------|------|
| `normal` (기본값) | CJK 문자를 단어 경계로 인정하지 않음 — 어절 중간에서도 끊김 |
| **`keep-all` (채택)** | **CJK 텍스트는 공백/구두점에서만 줄바꿈** — 어절 보존 |
| `break-all` | 모든 글자 사이에서 강제로 끊음 — 영문 단어도 분리 |

### Tailwind 클래스

Tailwind CSS v4는 `word-break: keep-all`에 대응하는 `break-keep` 유틸리티를 제공한다.

```tsx
// Before
<p className="font-light drop-shadow-lg max-w-3xl" style={quoteStyle}>

// After
<p className="font-light drop-shadow-lg max-w-3xl break-keep" style={quoteStyle}>
```

### 예상 결과

```
"만약 내 작품에 메시지가 들어 있다면 그것은
다름을 인정해도 좋다는 것이다. 그건 다른
말로 남과 달라도 괜찮다는 것, 외모가
다르고 행동이 다르고 말하는 방식도 다른
다른 색을 가진 누군가를 판단하기 앞서
스스로에게 물어야 한다는 것이다."
```

→ 어절(`다름을`, `남과`, `다르고`, `누군가를`, `것이다`)이 한 줄 안에 유지됨.

| 파일 | 변경 내용 |
|------|----------|
| `src/components/Quote.tsx` | 명언 본문 `<p>` 및 저자 `<p>`의 `className`에 `break-keep` 추가 |

## 4. 기대 효과

| 항목 | Before | After |
|------|--------|-------|
| 어절 보존 | 음절 단위 분리 (`다름을` → `다` / `름을`) | 어절 단위 보존 (`다름을`이 한 줄에 표시) |
| 가독성 | 단어가 끊겨 읽기 어려움 | 자연스러운 문장 흐름 |
| 시각적 완성도 | 단어 중간 끊김으로 어색함 | 명언 인용문에 어울리는 안정적 레이아웃 |
| 영문 명언 호환 | - | 영문은 기존과 동일하게 단어 단위로 줄바꿈 |

## 5. 리스크 및 고려사항

- **줄당 길이 변동**: `keep-all`을 적용하면 어절을 통째로 다음 줄로 넘기기 때문에, 줄당 글자 수가 들쭉날쭉해지고 우측 여백이 더 크게 보일 수 있음. 명언 인용 특성상 자연스러운 모양이므로 수용 가능.
- **공백 없는 매우 긴 토큰**: 공백 없이 매우 긴 문자열(예: 긴 URL, 합성어)이 들어오면 한 줄을 벗어날 수 있음. 현재 명언 콘텐츠에는 해당 사례 없음 → 발생 시 `overflow-wrap: anywhere` 보조 적용 검토.
- **저자명**: 저자명은 일반적으로 짧아 줄바꿈이 거의 발생하지 않으나, 일관성을 위해 동일하게 `break-keep` 적용.
- **로딩 스켈레톤**: 로딩 상태(`Quote.tsx:13-19`)는 텍스트가 없는 placeholder이므로 영향 없음.
